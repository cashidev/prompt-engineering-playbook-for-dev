## ⚡ Quick Start

### 1. Use a Proven Prompt Template

Instead of vague prompts like:

```
Improve this code
```

Use structured prompts:

```
You are a senior software engineer.

Task:
Refactor the following code for readability, performance, and maintainability.

Constraints:
- Do not change functionality
- Highlight trade-offs
- Suggest improvements with reasoning

Code:
<your code here>
```

👉 Result: **More reliable, production-quality output**

## 🧪 Example: Before vs After

### ❌ Bad Prompt

```
Fix this bug
```

### ✅ Good Prompt

```
You are a senior backend engineer.

Task:
Identify and fix the bug in the following code.

Requirements:
- Explain the root cause
- Provide a minimal fix
- Suggest how to prevent similar issues

Code:
...
```

## 🎯 Principles

* **Be explicit** → Clear instructions = better output
* **Add constraints** → Prevents hallucination
* **Iterate** → First output is rarely perfect
* **Verify** → Always review AI-generated code
